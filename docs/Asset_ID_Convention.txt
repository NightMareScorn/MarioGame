# QUY CHUẨN TÀI NGUYÊN (ASSETS) VÀ ID TILEMAP / OBJECT

## 1. Nguồn Tài Nguyên (Assets)
- **Quyết định:** Sử dụng bộ Asset của giáo viên cung cấp (hiện đã được load trong project).

## 2. Cấu Trúc File CSV (Map Format)
File CSV Map sẽ chia làm 3 phần chính để tách biệt nền (background) và các vật thể động (dynamic entities):
- `# SETTINGS`: Chứa cấu hình map (VD: cell_size,16). Có thể gán thêm `theme,underground` để báo game dùng gạch nền/chạc tĩnh màu tối.
- `# TILEMAP`: Chứa ma trận các ID lưới (grid) định nghĩa cảnh quan cố định.
- `# OBJECTS`: Chứa danh sách các đối tượng động (dynamic objects) và vật phẩm phụ.

## 3. Bảng Đồng Bộ Khóa ID Cho Tilemap
Cấu trúc ID sau sẽ được dùng trong phần `# TILEMAP` (dạng ma trận ô vuông) để parse Map và Code có chung tiếng nói:
- `0` = Trống (Empty / Background)
- `1` = Gạch nền (Ground Block)
- `2` = Mây (Cloud) - Nền
- `3` = Bụi cỏ (Bush) - Nền
- `4` = Gạch hỏi chấm (Question Block)
- `5` = Gạch cứng / bậc thang (Hard Block / Stair)
- `6` = Đồi tĩnh (Hill)
- `7` = Gạch đập được (Brick Block)
- `8` = Cột cờ (Flagpole)
- `9` = Lâu đài (Castle - Door/Body)
- `10` = Đồng xu tĩnh (Static Coin - dùng xếp chữ Warp Zone)
*(Ghi chú: Các ID tiếp theo sẽ được bổ sung tiếp vào phần này khi có thêm khối Tile mới, vd: ID dung nham, nước...)*

## 4. Định Dạng Cấu Trúc Object và Extra Properties
Để giải quyết luồng việc phân bố nhân vật và quái vật trong Game (hoặc các Object tự động tương tác), thống nhất sử dụng phần `# OBJECTS` như sau:
**Định dạng cơ bản:** `type, x, y, extra`
*(Trong đó x, y là tọa độ điểm thực tế tính bằng PIXEL hoặc WORLD UNIT, không phải tọa độ lưới trừ khi đã chia)*

**Quy chuẩn riêng cho Các Object:**

- **Người chơi (Mario):**
  `mario, <x_pixel>, <y_pixel>`

- **Kẻ thù / Quái vât:**
  - Goomba: `goomba, <x>, <y>, patrol_left=<giới_hạn_trái>, patrol_right=<giới_hạn_phải>`
  - Koopa (Rùa): `koopa, <x>, <y>, type=green_walking`
  - Buzzy Beetle (Bọ úp): `buzzy_beetle, <x>, <y>`
  - Piranha Plant (hoa ăn thịt đứng riêng): `piranha, <x>, <y>`

- **Vật phẩm (Items):**
  - Đồng xu (Coin): `coin, <x>, <y>` (nếu ở ngoài)
  - Đồng xu trong gạch: `coin, <x>, <y>, hidden_in_block=true`
  - Nấm (Mushroom): `mushroom, <x>, <y>, hidden_in_block=true`
  - Ngôi sao bạch kim (Star): `star, <x>, <y>, hidden_in_block=true`
  - Mạng 1-Up (1-Up Mushroom): `1-up, <x>, <y>, hidden_in_block=true, invisible_block=true`

- **Nền tảng di chuyển (Elevator/Moving Platform):**
  - Thang máy: `elevator, <x>, <y>, direction=up` (hoặc `down`, `left`, `right`)

- **Ống nước động (Pipe):**
  - Vừa đóng vai trò chướng ngại, vừa có thể tương tác:
  - Chỉ cản đường: `pipe, <x>, <y>, is_solid=true`
  - Có thể chui vào (để chuyển vào map ẩn vd `level_1_1_underground`):
    `pipe, <x>, <y>, dest_map=1_1_underground, enter_direction=down`
  - Ống chứa Piranha Plant:
    `pipe, <x>, <y>, has_piranha=true`
